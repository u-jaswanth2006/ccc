STEP 9 using CloudShell
1. Open CloudShell

Top right in AWS Console:

CloudShell icon (>_)

Click it.

Wait for terminal.

2. SSH into Bastion

Get Bastion public IP from EC2.

Run:

ssh ubuntu@BASTION_PUBLIC_IP

Example:

ssh ubuntu@44.197.xxx.xxx

If asked:

yes

If it says permission denied:
Use EC2 Instance Connect instead OR upload PEM to CloudShell.

Better/easiest method (CloudShell direct with PEM)

Upload both keys to CloudShell.

CloudShell → Actions → Upload file

Upload:

bastion.pem
dbserver.pem

Then:

chmod 400 bastion.pem
chmod 400 dbserver.pem

SSH to bastion:

ssh -i bastion.pem ubuntu@BASTION_PUBLIC_IP
3. Inside Bastion

Copy DB key from CloudShell to Bastion (new CloudShell tab):

scp -i bastion.pem dbserver.pem ubuntu@BASTION_PUBLIC_IP:~
4. Back inside Bastion

Fix permissions:

chmod 400 dbserver.pem

SSH to DB server:

ssh -i dbserver.pem ubuntu@DB_PRIVATE_IP

Example:

ssh -i dbserver.pem ubuntu@10.0.2.136
5. Proof

Inside DB:

hostname

Shows:

ip-10-0-2-xxx
